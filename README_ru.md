# Amnezia-WG Admin (awg-admin)

[English version](README.md)

**Amnezia-WG Admin** — приложение для централизованного управления инфраструктурой AmneziaWG и WireGuard. Оно хранит
конфигурацию серверов, пользователей и VPN-пиров и применяет её на управляемых серверах через агент, работающий на
каждой машине.

Один экземпляр приложения может управлять несколькими серверами одновременно. Конфигурация хранится в локальной базе
данных и является источником истины, а агенты приводят состояние серверов в соответствие с ней.

> 📖 **Установка, настройка и полное руководство:** [docs/GUIDE_ru.md](docs/GUIDE_ru.md)

## Обзор

Amnezia-WG Admin позволяет управлять инфраструктурой AmneziaWG и WireGuard без ручного редактирования конфигурационных
файлов и подключения к каждому серверу по SSH.

После добавления сервера приложение может автоматически развернуть на нём агент. Дальнейшее управление выполняется через
API агента по SSH-туннелю или напрямую по mTLS. При обнаружении расхождений между сохранённой конфигурацией и
фактическим состоянием сервера приложение повторно применяет необходимые изменения.

Пользователи и VPN-пиры являются отдельными сущностями. Каждый пользователь может владеть одним или несколькими пирами.
При создании пира автоматически генерируются ключи, назначаются адреса, применяются параметры AmneziaWG, формируются
клиентская конфигурация и QR-код.

Несколько серверов могут быть объединены в многоузловую VPN-топологию с маршрутизацией между узлами и использованием
выбранного сервера в качестве exit-ноды.

Приложение собирает информацию о состоянии управляемых серверов, включая загрузку процессора, использование памяти и
статистику WireGuard-пиров. Конфигурация инфраструктуры может быть экспортирована в переносимый формат и импортирована в
другую установку приложения.

Поддерживаются два режима работы: desktop-приложение и standalone веб-сервер. Оба используют одинаковую бизнес-логику,
модель данных и пользовательский интерфейс.

## Архитектура

Amnezia-WG Admin состоит из трёх компонентов: административного приложения, агента и пользовательского интерфейса.

### Admin app

Административное приложение хранит конфигурацию инфраструктуры, управляет серверами, пользователями и VPN-пирами, а
также координирует работу с агентами.

Приложение существует в двух вариантах: desktop-приложение на Wails и standalone веб-сервер. Оба используют общую
бизнес-логику и одинаковый пользовательский интерфейс, различаясь только способом запуска и транспортом взаимодействия с
frontend.

Исходный код расположен в корневом Go-модуле и каталоге `frontend/`.

### Agent

Агент представляет собой отдельный HTTP-сервис, запускаемый на каждом управляемом сервере.

Он управляет интерфейсами AmneziaWG и WireGuard, хранит их локальную конфигурацию и предоставляет API, используемое
административным приложением. Для взаимодействия с WireGuard используется библиотека `wgctrl-go`.

По умолчанию агент принимает подключения только на loopback-интерфейсе и рассчитан на работу через SSH-туннель. При
необходимости поддерживается прямое подключение по mTLS.

Исходный код агента расположен в каталоге `agent/`.

### Frontend

Пользовательский интерфейс реализован как SPA на React 19 и TypeScript.

При запуске frontend определяет среду выполнения и автоматически выбирает соответствующий транспорт: Wails API для
desktop-приложения или HTTP API standalone-сервера. Пользовательский интерфейс остаётся одинаковым в обоих режимах.

Исходный код расположен в каталоге `frontend/`.

### Хранение данных

Все метаданные инфраструктуры хранятся в локальной базе данных BoltDB (`~/.awg-admin`).

База содержит сведения о серверах, SSH-подключениях, пользователях, VPN-пирах и их взаимосвязях и является единственным
источником истины для административного приложения.

Агент хранит только локальную конфигурацию интерфейсов и не содержит информации о структуре инфраструктуры в целом.

## Сборка из исходного кода

Для сборки проекта требуются Go **1.26.2** или новее и Node.js **24** или новее.

Доступны следующие цели `Makefile`:

```sh
make server      # standalone веб-сервер
make run-server  # локальный запуск через go run
make desktop     # desktop-приложение на Wails
make agent       # сборка Linux-агента
make migrate     # утилита awg-migrate
```

Артефакты сборки помещаются в каталог `build/bin/`. Команда `make run-server` предназначена для локальной разработки и
запускает приложение без создания исполняемого файла.

Для сборки desktop-приложения требуется установленный Wails CLI:

```sh
go tool wails
```

При выполнении целей, использующих пользовательский интерфейс, frontend собирается автоматически, поэтому отдельный
запуск инструментов Node.js обычно не требуется.
